## 1. Vision

A web app (deployed on **Vercel**) that helps a buyer in the **Singapore property
market** make data-driven decisions on whether to **buy a resale property** or a
**new launch (Building-Under-Construction / TOP)** unit, and to model the
**cashflow** of holding the property over an N-year horizon — comparing
**self-stay vs. rent-out** outcomes.

The core value is a **transparent, adjustable financial model**: every assumption
(appreciation rate, rental growth, interest rate, holding period) is a slider/input,
and the projection updates live.

## 2. Target Market & Domain Assumptions (Singapore)

These rules drive the calculators. They are encoded as configurable constants so
they can be updated when policy changes.

### 2.1 Loan-to-Value (LTV) & Down Payment
- 1st housing loan: **LTV 75%** → **25% down** (min **5% cash**, remainder cash/CPF).
- Max loan tenure: **30 years** (private), capped so tenure + age ≤ 65 for best LTV.

### 2.2 Buyer's Stamp Duty (BSD) — residential, tiered on purchase price
| Band | Rate |
|------|------|
| First $180,000 | 1% |
| Next $180,000 | 2% |
| Next $640,000 | 3% |
| Next $500,000 ($1.0m–$1.5m) | 4% |
| Next $1,500,000 ($1.5m–$3.0m) | 5% |
| Above $3,000,000 | 6% |

### 2.3 Additional Buyer's Stamp Duty (ABSD) — by profile (2023 rates)
| Profile | 1st | 2nd | 3rd+ |
|---------|-----|-----|------|
| SG Citizen | 0% | 20% | 30% |
| SG PR | 5% | 30% | 35% |
| Foreigner | 60% | 60% | 60% |

### 2.4 TDSR
- Total Debt Servicing Ratio cap: **55%** of gross monthly income.
- Used as an affordability check (stress-test rate **4%** for the calc).

### 2.5 New Launch (BUC) Progressive Payment Schedule
Down payment + stage payments tied to construction milestones until **TOP** and CSC:
| Stage | % of price |
|-------|-----------|
| Booking (OTP) | 5% (cash) |
| S&P signing (within 8 wks) | 15% |
| Foundation | 10% |
| Reinforced concrete framework | 10% |
| Partition walls | 5% |
| Roofing/ceiling | 5% |
| Door/window frames | 5% |
| Carpark/roads/drains | 5% |
| **TOP** (Temporary Occupation Permit) | 25% |
| CSC (legal completion) | 15% |

> Note: rates above are defaults; the UI exposes them so users can override per project.

## 3. User Stories

### User Story 1 — Resale Property (immediate ownership)
> As a buyer, I want to buy a **resale** property starting on a chosen date,
> project **X years** forward, toggle between **self-stay vs. rent-out**, and adjust
> **annual capital growth %** and **rental growth %**, so I can see the projected
> equity, cashflow and net position.

Inputs:
- Purchase date (DD/MM/YYYY)
- Purchase price
- Down payment % (default 25%), interest rate %, loan tenure
- Buyer profile (Citizen/PR/Foreigner) + property count (drives ABSD)
- Mode: **Self-stay** | **Rent-out**
- If rent-out: monthly rental, rental growth % p.a., vacancy %, maintenance %
- Annual capital appreciation %
- Projection horizon (N years)

Outputs:
- Year-by-year table: property value, outstanding loan, equity, monthly mortgage,
  net rental cashflow, cumulative cashflow.
- Total cash-in (down payment + BSD + ABSD + fees).
- Self-stay vs rent-out comparison (opportunity cost of imputed rent).
- IRR / total ROI at end of horizon.

### User Story 2 — New Launch (BUC, TOP in the future)
> As a buyer, I want to buy a **new project that TOPs in MM/YYYY** (today is MM/YY),
> see the **current down payment needed**, **estimated property price at TOP**,
> the **expected rental** at TOP, and the **growth potential N years** after TOP.

Inputs:
- Today (MM/YYYY) and **TOP date** (MM/YYYY)
- Purchase price (today)
- Pre-TOP appreciation % p.a. (launch → TOP)
- Progressive payment schedule (defaults from §2.5, editable)
- Estimated rental at TOP, rental growth %
- Post-TOP appreciation % p.a.
- Projection horizon (N years after TOP)

Outputs:
- Cash outlay timeline (each progressive milestone, when due, cumulative).
- **Current down payment needed** (booking + S&P + BSD/ABSD).
- **Estimated price at TOP** (compounded appreciation).
- Estimated rental yield at TOP.
- N-year post-TOP projection (value, equity, rental cashflow).

## 5. Architecture

### 5.1 Tech Stack
- **Next.js (App Router)** + **TypeScript**
- **Redux Toolkit (RTK)** for state, **redux-persist** (localStorage) for persistence
- **Tailwind CSS** for styling (UI guided by `ui-ux-pro-max` skill)
- **Recharts** for projection charts
- Deployed on **Vercel**

### 5.2 Folder Structure
```
src/
  app/                      # Next.js App Router routes
    layout.tsx
    page.tsx                # landing / tool hub
    providers.tsx           # Redux + persist provider (client)
    resale/page.tsx         # User Story 1
    new-launch/page.tsx     # User Story 2
    tools/
      down-payment/page.tsx
      stamp-duty/page.tsx
      mortgage/page.tsx
  store/
    index.ts                # configureStore + persist
    hooks.ts                # typed useAppSelector/useAppDispatch
    slices/
      resaleSlice.ts        # Story 1 inputs
      newLaunchSlice.ts     # Story 2 inputs
      settingsSlice.ts      # buyer profile, global assumptions
  lib/
    finance/
      stampDuty.ts          # BSD + ABSD
      mortgage.ts           # amortization
      projection.ts         # year-by-year cashflow engine
      progressivePayment.ts # BUC schedule
      constants.ts          # SG policy defaults (§2)
      types.ts
  components/
    ui/                     # buttons, inputs, sliders, cards
    charts/
    forms/
```

### 5.3 Pure Calculation Core
All finance math lives in `src/lib/finance/*` as **pure, unit-testable functions**
with no React/Redux dependency. RTK slices hold *inputs only*; derived projections
are computed via selectors / memoized helpers. This keeps the model transparent and
testable independent of the UI.

### 5.4 Data Flow
```
User input → RTK slice (inputs) → selector → lib/finance pure fns → derived projection → chart/table
                     ↓
            redux-persist → localStorage
```
